# 学生运动与成长数据驾驶舱

## 一句话说明

本系统不是简单记录学生有没有运动，而是把 **运动、作业、睡眠、成绩、心理状态、家庭支持和学校环境** 放在同一个数据视角里，帮助学校判断：

> 学生在什么学习压力、睡眠状态和运动频率下，成长状态最好。

## 我们想解决的问题

当前学校管理体育与学生成长时，常见问题是：

1. 只知道学生是否完成运动打卡，不知道运动是否真正有效。
2. 只看体育成绩，不知道作业压力、睡眠、情绪是否影响运动参与。
3. 只看学业成绩，不知道适度运动是否帮助学生保持状态。
4. 管理层看到的是零散数据，缺少全校、年级、班级之间的趋势对比。
5. 班主任、体育老师、年级主任、校长看到的数据口径不统一。

本系统的目标是把这些分散数据变成一个清晰的管理驾驶舱。

## 管理层最终能看到什么

Top level 管理层可以看到 12 大类数据：

| 类别 | 管理层能回答的问题 |
|---|---|
| 学生基础维度 | 学生结构、班级分布、体质基础、特殊运动限制是否清楚？ |
| 运动参与度 | 哪些年级、班级、学生运动参与不足？ |
| 运动质量 | 学生运动是否达到有效强度？是否存在疲劳和伤病风险？ |
| 作业与学习压力 | 作业量是否挤压学生运动和睡眠？ |
| 运动积极度 | 学生是主动运动，还是只完成最低要求？ |
| 学业成绩 | 运动参与变化后，成绩趋势是上升、下降还是稳定？ |
| 睡眠与恢复 | 学生睡眠是否支持运动和学习恢复？ |
| 心理与情绪 | 运动是否改善情绪、自信和学习压力？ |
| 家庭与环境 | 家长、通勤、补习、场地是否影响学生运动？ |
| 学校与班级环境 | 体育课、班级氛围、场地使用是否支持运动？ |
| 关系型指标 | 作业量、睡眠、运动、成绩之间有什么关系？ |
| 高级指数 | 用综合指数快速判断全校、年级、班级状态。 |

## 最核心的管理价值

### 1. 看趋势，而不是只看结果

系统会用折线图查看：

- 按周趋势
- 按月趋势
- 按学期趋势

管理层可以快速看出：

- 运动参与是否在下降
- 考试前运动量是否明显减少
- 某个年级是否持续落后
- 某个班级是否突然出现异常

### 2. 看关系，而不是只看单项数据

系统重点分析这些关系：

| 关系 | 管理意义 |
|---|---|
| 作业量 vs 运动参与 | 判断作业负担是否压缩体育活动 |
| 睡眠不足 vs 运动积极度 | 判断学生是否因为恢复不足不愿运动 |
| 运动参与 vs 成绩变化 | 判断适度运动是否帮助学习状态稳定 |
| 考试周 vs 运动量 | 判断考试压力是否导致运动中断 |
| 家长支持 vs 持续运动 | 判断家庭环境是否影响运动习惯 |
| 班级氛围 vs 个人参与 | 判断班级管理是否带动学生运动 |

### 3. 看指数，而不是让领导看一堆原始表格

系统把复杂数据汇总成 10 个核心指数：

| 指数 | 含义 |
|---|---|
| 运动参与指数 | 学生是否经常运动、持续运动、达标运动 |
| 运动积极度指数 | 学生是否主动、有兴趣、有挑战意愿 |
| 学习负担指数 | 作业、补习、迟交、压力和睡眠压缩程度 |
| 睡眠恢复指数 | 睡眠时长、质量、困倦和运动后恢复情况 |
| 学业变化指数 | 成绩、排名、作业正确率、小测和注意力趋势 |
| 情绪压力指数 | 压力、焦虑、疲惫、心情和自信状态 |
| 健康风险指数 | BMI、伤病、运动限制、疲劳和心率异常风险 |
| 班级活跃指数 | 一个班级整体运动氛围和参与水平 |
| 年级体育发展指数 | 一个年级整体体育发展水平和均衡度 |
| 学生综合成长指数 | 综合运动、学习、睡眠、心理和健康状态 |

## 数据从哪里来

为了减少老师和学生负担，数据采集分成 7 种方式：

| 代码 | 采集方式 | 举例 |
|---|---|---|
| A | 系统自动生成 | 打卡时间、项目、次数、达标率、上传率、审核率 |
| B | 学校系统导入 | 学生档案、成绩、课表、作业量、体测数据 |
| C | 教师录入/评价 | 请假、课堂表现、体育课占用、伤病记录 |
| D | 学生轻量问卷 | 疲劳、压力、睡眠、兴趣、运动后感受 |
| E | 家长问卷 | 家庭支持、补习、通勤、周末运动、作息 |
| F | 设备/传感器 | 心率、步数、睡眠、配速、运动强度 |
| G | 模型计算 | 指数、趋势、相关性、风险预警 |

## 哪些数据优先做

### 第一阶段：先做低风险、低成本数据

优先采集：

- 学生、年级、班级
- 项目、打卡、成绩、达标
- 全校、年级、班级、学生个人趋势
- 运动参与指数
- 班级活跃指数
- 年级体育发展指数

价值：

> 快速建立体育管理基础盘，让领导马上看到全校运动参与情况。

### 第二阶段：加入学习压力和成绩

加入：

- 作业量
- 作业完成时间
- 学业成绩
- 排名变化
- 小测和作业正确率

价值：

> 开始分析“作业压力是否影响运动”和“运动是否影响学习状态”。

### 第三阶段：加入睡眠、心理和家庭环境

加入：

- 睡眠时长
- 疲劳程度
- 学习压力
- 运动兴趣
- 家长支持
- 补习时间

价值：

> 从单纯体育管理升级为学生综合成长管理。

### 第四阶段：加入设备和高级预测

加入：

- 心率
- 步数
- GPS/配速
- 睡眠设备数据
- 风险预警
- 关系预测

价值：

> 形成精细化、个性化、长期趋势管理。

## 指数怎么计算

系统不是人工主观打分，而是把多个数据加权计算成指数。

例如：

### 运动参与指数

由这些数据组成：

- 参与率
- 达标率
- 月运动次数
- 连续运动天数
- 缺席率

管理意义：

> 判断学生是否真正形成稳定运动习惯。

### 学习负担指数

由这些数据组成：

- 每日作业时长
- 补习时长
- 作业迟交率
- 作业压力自评
- 睡眠被作业压缩比例

管理意义：

> 判断学习压力是否已经影响运动、睡眠和情绪。

### 学生综合成长指数

由这些指数综合得到：

- 运动参与指数
- 运动积极度指数
- 睡眠恢复指数
- 学业变化指数
- 班级活跃指数
- 年级体育发展指数
- 学习负担指数
- 情绪压力指数
- 健康风险指数

说明：

- 正向指数越高越好。
- 学习负担、情绪压力、健康风险是反向指标。
- 综合成长指数不是简单平均，而是加权综合。

## 权限设计

| 权限 | 能看到的数据 |
|---|---|
| Top level：校长/体育主管 | 全校数据、年级数据、班级数据、学生个人数据、12 大类指标、全部指数 |
| Medium level：年级主任 | 本年级数据、班级数据、学生个人数据 |
| Basic level：班主任/体育老师 | 授权班级数据、学生个人数据 |
| Student：学生 | 只能看自己的数据 |

## 这个系统最终帮助学校做什么决策

管理层可以用它做这些判断：

1. 哪个年级运动参与下降，需要干预。
2. 哪个班级作业压力过高，影响运动和睡眠。
3. 哪些学生只是完成最低要求，没有形成运动积极性。
4. 哪些班级运动氛围好，值得推广经验。
5. 考试前后运动量变化是否过大。
6. 适度运动是否帮助学生成绩和情绪更稳定。
7. 哪些学生存在疲劳、伤病、低动力风险。
8. 学校体育管理是否真正带来长期成长。

## 对领导最重要的一句话

这个系统的价值不是“多收集数据”，而是：

> 用运动数据作为入口，建立一个能看见学生学习压力、身体状态、心理状态和成长趋势的学校管理驾驶舱。

## 当前系统已实现的方向

目前统计页已经具备：

- Top level 全域管理驾驶舱
- 12 大类指标切换查看
- 采集形式说明按钮
- 指数计算函数说明
- 周/月/学期趋势折线图
- 权限分层查看
- 全校、年级、班级、学生个人数据层级

下一步可以继续强化：

- 把问卷、成绩、作业、睡眠数据接入真实数据源
- 为每个指数显示历史趋势
- 增加异常预警
- 增加班级/年级对比
- 输出校级月报和年级周报
